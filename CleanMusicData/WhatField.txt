# Data "FMA: A Dataset For Music Analysis" 

- Fichier csv "features.csv" : Données provenant de la librairie python "librosa" qui analyse le spectre des chansons donc pas besoin dans notre étude ! (supprimé dans le dossier d'origine sur github car dossier très lourd)

- Fichier csv "tracks.csv" : Contient les informations sur les chansons, leurs date de creation, poducteurs,compositeur, la localisation (ville) avec longitude/latitude et le genre 

- Fichier csv "raw_tracks.csv" : Ressemble beaucoup au fichier "tracks" il contient en plus des informations sur l'id du track, le nom de l'artiste et le genre du track

- Fichier csv "raw_genres.csv" : Ressemble beaucoup au fichier "genres" donc pas besoin non plus !

- Fichier csv "genres.csv" : Pas de lien de jointure possible

- Fichier csv "raw_albums.csv" : Contient les informations des albums tel que le titre, date de création .. Aucune information ne semble être intéressant pour notre étude

- Fichier csv "raw_artists.csv" : Contient les informations sur les artistes. Les informations qui peuvent être utiles sont : le nom de l'artiste, la localisation (longitude, latitude). Après avoir fait des recherches sur cette base de données les artistes connus : Drake, Dua Lipa, Taylor Swift ... on s'est apperçu qu'ils n'étaient pas présent. Il faudrait regarder sur une autre base de données pour avoir tous les artistes. 

- Fichier csv "raw_echonest.csv" : Contient pas mal d'informations sur une musique identifiable avec "track_id". Faut donc utiliser le fichier "track.csv" pour connaître de quelle musique il s'agit.
Comme pour le fichier "raw_artists", les artistes connus ne sont pas présents.

=> Nous pensons qu'il ne sera pas possible d'effectuer de jointures entre la table "universal_top_spotify_song" et FMA
